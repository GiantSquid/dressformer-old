############# Макет встраиваемого приложения "Dressformer" #############

Задачи:

1. Масштабируемость. Должно быть достаточно места под новый функционал. При добавлении новых функций должна быть ясна логика их представления.
2. Тач-дизайн. Во все UI-элементы должно быть просто тыкать пальцами, во всех UI-элементах должны работать тач-события.
3. Прозрачность. Приложение не должно полностью перекрывать материнскую страницу, у пользователя не должно создаваться впечатления, что он попал на другой сайт.
4. Легкость. Функциональные блоки не должны быть перегружены, приложение не должно выглядеть сложным из-за обилия и плотности элементов управления.
5. Минимализм. Все необязательные функции должны быть исключены.
6. Сочетаемость. Интерфейс не должен содержать дизайнерских изысков, резко выделяющих встраиваемый модуль в визуальной среде материнского сайта. Нужно использовать универсальные цветовые схемы из градаций серого.
7. Кастомизация. Необходимы элементы, позволяющие при минимуме манипуляций с интерфейсом, решать задачи брендирования и визуального соответствия с материнским сайтом.
8. Интернациональность. Подписи на различных языках должны помещаться без переносов. У всех UI-элементов должны быть иконки, помогающие ориентироваться без знания текущего языка.
9. Акценты. Элементы, реализующие ключевые пользовательские действия, должны быть хорошо акцентированы и соответствовать устоявшимся паттернам.
